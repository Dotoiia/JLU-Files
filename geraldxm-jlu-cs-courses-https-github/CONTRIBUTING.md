# 贡献指南

感谢你愿意补充课程资料。为了让仓库长期可维护，请尽量遵守下面的约定。

## 提交前检查

- 确认资料可以公开分享。
- 删除姓名、学号、账号、群聊记录、密钥等个人或敏感信息。
- 如果资料来自他人，请注明来源和授权情况。
- 如果课程内容有年份差异，请在文件名或文档开头写明适用年份。

## 推荐 PR 格式

```markdown
## 内容

- 添加/更新了哪些课程资料

## 来源

- 自己整理 / 公开资料链接 / 已获授权

## 备注

- 是否有版本、教师、年份或适用范围限制
```

## 目录约定

- 每门课程保留一个 `README.md`，用于说明该目录有什么资料。
- 课程资料优先放入 `资料/`。
- 实验、作业、环境配置和踩坑记录优先放入 `作业与实验/`。
- 大文件建议先压缩或改用外部链接，并在 README 中说明。

## 内容风格

- Markdown 文档使用清晰标题和列表。
- 代码块注明语言。
- 图片尽量放在同级 `assets/` 目录中。
- 不确定的信息请标注“可能过期”或“待核对”。
